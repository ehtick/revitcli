## Purpose

Add `revitcli workflow registry` as a read-only local registry over governed
workflow YAML files for the v6.0 Local BIMOps Workbench.

This is not a workflow runner, Revit writer, ledger replay engine, database,
SaaS control plane, MCP adapter, built-in LLM parser, or dashboard-central
state source.

## Notes

- `workflow registry` indexes local workflows; it does not execute workflow
  steps or create workflow-run receipts.
- Missing default workflow directories are warning-only so a fresh project can
  inspect the registry contract before installing templates.
- Review fixes integrated: new workflow-facing command families now validate
  grouped subcommands, read-only declarations warn for write-capable planners,
  registry risk/scope escalates misdeclared write-capable read-only steps, and
  workbench/release gates exercise a real `workflow-registry.v1` payload.
- Parity follow-up integrated: registry tests and the workbench runtime gate now
  assert delivery bundle, schedule export, publish output, `journal verify`,
  package receipt schemas, publish receipt schemas, and
  `schedule-export-manifest.v1` inference without executing workflow steps.
- Review follow-up integrated: registry inputs now include selected concrete
  local references such as `profile:...`, `manifest:...`, `plan:...`, and
  `receipt:...`; `journal verify` acceptance evidence is inferred only when the
  workflow explicitly declares a journal verify command.
- Runtime evidence follow-up integrated: the workbench gate now executes JSON,
  table, and Markdown registry output against the same fixture, checks
  `JSON/table/Markdown output semantic parity`, and requires final file-tree
  snapshot plus event-level no-write evidence.
