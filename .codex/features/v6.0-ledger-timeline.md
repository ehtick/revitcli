## Purpose

Add `revitcli ledger timeline` as a read-only project-memory timeline over
local operation ledger artifacts.

This is not a ledger database, replay engine, cross-project analytics service,
dashboard state source, or agent decision layer.

## Notes

- `ledger timeline` summarizes existing local artifacts; it does not create a
  persistent timeline store.
- Invalid or missing operation timestamps are reported as unbucketed timestamp
  warnings instead of being coerced into a false bucket.
- `report knowledge` remains the broader project report; `ledger timeline` is
  the ledger-specific deterministic time view used by v6.0 project memory.
- Cross-project local timeline follow-up integrated: repeated `--project` roots
  provide local multi-project `projectDirectories` and `byProject` evidence
  without a database, cloud sync, or dashboard state source.
- Review fixes integrated: `ledger-timeline.v1` now exposes top-level and
  per-bucket `issuesBySeverity`, runtime gate evidence covers journal, history,
  delivery, and workflow sources, and both focused tests and the workbench gate
  assert no-write behavior with local tree snapshots.
- Receipt-status parity follow-up integrated: focused tests now assert
  `ledger timeline --receipt-status missing` filters before bucketing and keeps
  top-level and bucket-level missing-receipt and issue-severity summaries in
  sync with the existing query filter contract, while excluding valid receipt
  operations from the filtered bucket.
- Category/operator parity follow-up integrated: `ledger-timeline.v1` buckets
  now include `byCategory` and `byOperator` summaries, and table/Markdown output
  plus workbench/release gates document category counts per bucket and operator
  counts per bucket.
- Timestamp semantics follow-up integrated: timestamp bucketing now requires
  ISO 8601 values with an explicit UTC offset (`Z` or `+/-HH:mm`), and
  offset-less timestamps are reported as unbucketed warnings.
- Timestamp filter follow-up integrated: focused tests and the v6.0 workbench
  runtime gate now assert `--since` and `--until` preserve unbucketed timestamp
  warnings instead of hiding malformed or offset-less records during timeline
  review.
- Output-format no-write parity integrated: focused tests now assert JSON,
  Markdown, and table timeline output leave the local artifact tree unchanged,
  including empty-directory regressions.
- Output semantic parity follow-up integrated: focused tests, workbench runtime
  evidence, release gates, and smoke docs now assert JSON/table/Markdown
  timeline semantic parity for bucket, operation, issue, source, action,
  category, operator, receipt-status, and issue-severity summaries, with final
  file-tree snapshot evidence and event-level no-write evidence.
