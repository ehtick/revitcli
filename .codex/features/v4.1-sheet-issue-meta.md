## Pre-flight

Before the first tick on a fresh `.codex/` setup, a human must decide what
to do with the v4.1 work that is already uncommitted in this repository
(SheetIssuePlanner.cs, the SheetsCommand issue-meta subcommand, PlanCommand
sheet-issue branches, and the new test files). Two options:

1. **Commit as baseline** — preferred:
   ```
   git add src/RevitCli/Sheets src/RevitCli/Commands tests/RevitCli.Tests/Commands ...
   git commit -m "wip(v4.1): scaffold sheet issue meta — pre-tick baseline"
   ```
   Then this checklist starts from a clean working tree.
2. **Discard and let the loop rebuild** — destructive: `git stash` or
   `git checkout --`. Not recommended for first runs.
