## Purpose

Move room and door/window numbering from reviewed plan surface toward controlled
apply pilots. This is intentionally later than sheet and schedule closure
because numbering rules are office-specific and can cause high-cost downstream
errors. v5.0 RC is now GO for a Revit 2026-only live claim; v5.1/v5.2 live
fixtures remain pending, so this slice proceeds by explicit written decision as
portable numbering hardening only.

## Commands

- `rooms renumber`
- `marks assign`
- `marks verify`
- `plan show`
- `plan apply`
- `rollback`
- `journal verify`

## Notes

- Do not make numbering a default automatic fix. It must stay dry-run first,
  plan reviewed, approval gated, receipt backed, and rollbackable.
- Do not use an LLM, MCP, SaaS service, or dashboard as the source of numbering
  decisions. Those can only call or view deterministic CLI artifacts later.
