## Purpose

Add the first runtime surface for the Revit Model Operations Ledger: a
terminal-first, local-first, deterministic, read-only `ledger query` command
that indexes existing local artifacts.

This is not a Revit mutation feature and not a centralized ledger runtime.

## Notes

- Missing artifact roots are warnings; malformed artifacts become structured
  issues and do not make the command write or crash.
- Review fixes integrated: malformed delivery/workflow issue-only operations no
  longer double-count issues, malformed workflow receipt issues are sorted by
  path/message, and shell completions plus the workbench completion gate cover
  ledger query subcommands, options, sources, receipt statuses, and output
  formats.
- `ledger query` can support future project-memory views, but this slice does
  not add replay, cross-project analytics, a database, a dashboard source of
  truth, or agent decision logic.
- Parity follow-up integrated: focused tests and the v6.0 workbench runtime
  gate now assert `ledger query` emits `ledger-query.v1` across journal,
  history, delivery, and workflow sources while leaving the local artifact tree
  unchanged.
- Determinism follow-up integrated: focused tests now lock same-source,
  same-timestamp journal ordering by artifact line number instead of relying on
  incidental input order.
- Runtime determinism follow-up integrated: the v6.0 workbench runtime gate now
  uses a same-timestamp journal/history/delivery/workflow fixture and asserts
  exact timestamp/source/path/line ordering in `ledger-query.v1`.
- Runtime no-write follow-up integrated: the v6.0 workbench runtime gate now
  combines final file-tree snapshots with event-level no-write evidence for
  `ledger query` and the paired `ledger validate` path.
- Output-format no-write parity integrated: focused tests now assert JSON,
  Markdown, and table query output leave the local artifact tree unchanged,
  including empty-directory regressions.
- Output-format semantic parity follow-up integrated: focused tests and the v6.0
  workbench runtime gate now assert JSON/table/Markdown output preserve the same
  operation count, ordering, timestamp, source, action, receipt status, and
  artifact path fields.
- Timestamp filter follow-up integrated: focused tests now assert time-bound
  query output drops invalid-timestamp operations instead of mixing malformed or
  offset-less records into filtered result sets.
