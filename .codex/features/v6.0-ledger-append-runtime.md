## Purpose

Add the first write-capable local runtime slice for the Revit Model Operations
Ledger: `revitcli ledger append` creates reviewed local JSONL records only when
explicitly approved with `--yes`.

This is not a live Revit mutation feature, not ledger replay, and not a
database-backed operations runtime.

## Notes

- This slice intentionally stops before replay/apply, database storage,
  cross-project analytics, and broader live Revit ledger integration beyond
  recording successful approved `set` writes and receipt-backed `export`
  operations plus manifest-backed `schedules batch-export` rows.
- `ledger append` records can reference receipts, artifacts, model identity,
  plan hash, rollback pointer, and evidence links; command-specific receipt
  schemas remain the source of truth for actual operation details.
- Gate binding is now explicit: v6.0 fails if append runtime evidence, append
  docs, or `ledger-append.v1` contract exposure are missing.
- `ledger-operation.v1` now emits the required v6 audit fields listed in the
  contract, including operation identity, command/args, working directory,
  timing, risk/approval metadata, journal path, checks, artifacts, and optional
  affected element evidence.
