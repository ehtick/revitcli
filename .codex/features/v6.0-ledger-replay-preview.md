# v6.0 Ledger Replay Preview and Bounded Apply

Status: complete

Add a portable default-preview `revitcli ledger replay` slice over appended
local ledger records, plus a narrow live apply path for successful approved
`set --yes` source-ledger records with frozen affected ids and successful
receipt-backed `export` source-ledger records with recorded args, plus
successful manifest-backed `schedules.batch-export` records with recorded CSV
output entries.

- [x] **command-surface** - `ledger replay --source ledger --output json`
  emits `ledger-replay.v1`.
  - evidence: focused `LedgerCommandTests` cover JSON preview output, invalid
    limit rejection, and file-hash no-write behavior.
- [x] **contract-binding** - Workbench and release gates require the command
  path, output schema, smoke doc, and runtime evidence.
  - evidence: `v60LocalBimOpsContractGate.runtimeEvidence.ledgerReplay` is
    required by `release verify --strict`.
- [x] **bounded-apply** - `ledger replay --source ledger --action set --apply
  --yes --output json` applies only eligible approved set records with frozen
  affected ids.
  - evidence: focused `LedgerCommandTests` cover request construction and
    approval/source guards, failed/blocked no-audit-write behavior, and
    non-replayable replay-audit rows with best-effort
    `modelIdentity`/`modelPath`/`revitVersion` evidence when status is
    available;
    `.artifacts/live-smoke/revit2026-v6-ledger-replay-apply-audit-20260525-121641.json`
    records one controlled Revit 2026 live apply with `appliedStepCount=1`
    and one `ledger.replay.apply` audit row; the Revit smoke scripts now parse
    the isolated ledger file and fail if the replay-audit row is missing
    affected-id, replay-arg, or live model/version evidence.
- [x] **bounded-export-apply** - `ledger replay --source ledger --action
  export --apply --yes --output json` applies only successful receipt-backed
  export records with recorded `--format`, selector, and `--output-dir` args.
  - evidence: focused `LedgerCommandTests` cover export request
    reconstruction and missing-output-dir blocking; live Revit 2026 smoke under
    `.artifacts/live-smoke/revit2026-v6-export-replay-apply-20260525/`
    records one successful PDF export row and one non-replayable
    `ledger.replay.apply` audit row with live model/version evidence.
- [x] **bounded-schedules-apply** - `ledger replay --source ledger --action
  schedules.batch-export --apply --yes --output json` applies only successful
  manifest-backed schedule batch-export records with recorded CSV output
  entries.
  - evidence: focused `LedgerCommandTests` cover schedule export request
    reconstruction, CSV writeback, replay-audit append, and missing-manifest
    blocking.
- [x] **docs** - README, roadmap, v6 contract, gap report, and smoke docs scope
  replay as preview by default with bounded approved set/export/schedule apply.
  - evidence: `docs/smoke/v6.0/ledger-replay.md` states default `dryRun=true`,
    no writes/no Revit start for preview mode, and explicit `--apply --yes`
    limits.

Out of scope:

- Applying non-`set`/`export`/`schedules.batch-export` ledger records to Revit.
- Applying unapproved records or records without frozen affected ids.
- Database-backed ledger replay.
- Cross-project analytics.
- Office rollout evidence.
