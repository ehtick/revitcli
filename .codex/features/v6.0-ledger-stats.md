## Purpose

Add `revitcli ledger stats` as a read-only project-memory summary over local
operation ledger artifacts.

This is not a ledger database, replay engine, cross-project analytics service,
dashboard state source, or agent decision layer.

## Notes

- `ledger stats` summarizes existing local artifacts; it does not create a
  persistent stats store.
- Repeated `--project` roots provide local cross-project analytics without a
  database, cloud sync, or dashboard state source.
- `report knowledge` remains the broader project report; `ledger stats` is the
  ledger-specific deterministic summary used by v6.0 project memory.
- Review fixes integrated: table and Markdown output now include category,
  operator, and issue-severity summaries; workbench and release gates require
  those documented dimensions; tests cover the JSON and human-readable surfaces.
- Parity follow-up integrated: focused tests and the v6.0 workbench runtime
  gate now assert malformed journal/delivery/workflow artifacts are summarized
  as issue source and issue severity counts and that `ledger stats` leaves the
  local artifact tree unchanged.
- Receipt-status parity follow-up integrated: focused tests now assert
  `ledger stats --receipt-status missing` reuses the same query filter contract
  as `ledger query` and `ledger validate`, including missing-receipt counts and
  issue source/severity summaries, while excluding valid receipt operations from
  the filtered summary.
- Output-format no-write parity integrated: focused tests now assert JSON,
  Markdown, and table stats output leave the local artifact tree unchanged,
  including empty-directory regressions.
- Output semantic parity follow-up integrated: focused tests, workbench runtime
  evidence, release gates, and smoke docs now assert JSON/table/Markdown stats
  semantic parity for summary counters and source/action/category/operator/
  receipt-status/issue-source/issue-severity count buckets, with final
  file-tree snapshot evidence and event-level no-write evidence.
- Timestamp filter follow-up integrated: focused tests now assert time-bound
  stats output drops invalid-timestamp operations before summary grouping.
