## Purpose

Add `revitcli ledger validate` as the read-only integrity check for the local
Revit Model Operations Ledger artifact graph.

This is not a Revit mutation feature, not an artifact repair feature, and not a
centralized ledger runtime.

## Notes

- `ledger validate` does not repair missing artifacts or receipts.
- `--fail-on error` fails on validation errors; `--fail-on warning` also fails
  on warnings.
- Review fixes integrated: missing or unreadable receipts now always emit
  `receipt.missing` / `receipt.unreadable` so `receipt-status` cannot pass
  falsely; explicit missing sources fail as `source.missing`; workbench and
  release gates require source-readability coverage in the v6.0 smoke docs.
- This slice strengthens local project-memory evidence without adding replay,
  cross-project analytics, a database, dashboard source of truth, or agent
  decision logic.
- Parity follow-up integrated: focused tests and the v6.0 workbench runtime
  gate now assert `ledger validate` emits `ledger-validate.v1` with source
  readability, artifact link, receipt status, timestamp-format, and no-write
  evidence, including a failing validation no-write path.
- Runtime no-write follow-up integrated: the v6.0 workbench runtime gate now
  combines final file-tree snapshots with event-level no-write evidence for
  `ledger validate` and its paired `ledger query` path.
- Timestamp follow-up integrated: focused tests now assert `timestamp.invalid`
  is emitted with warning severity, source/path/line context, and
  `--fail-on warning` failure behavior.
- Timestamp semantics follow-up integrated: timestamp parsing now requires
  ISO 8601 values with an explicit UTC offset (`Z` or `+/-HH:mm`), and
  offset-less timestamps are reported as warnings instead of being silently
  coerced.
- Timestamp filter follow-up integrated: focused tests and the v6.0 workbench
  runtime gate now assert `--since` and `--until` preserve invalid timestamp
  warnings instead of hiding malformed or offset-less records during validation.
- Output-format no-write parity integrated: focused tests now assert JSON,
  Markdown, and table validation output leave the local artifact tree unchanged
  even when validation fails, including empty-directory regressions.
- Output semantic parity follow-up integrated: focused tests, workbench runtime
  evidence, release gates, and smoke docs now assert validation JSON/table/
  Markdown semantic parity for valid status, counts, check ids, check statuses,
  and check evidence.
- Receipt-hash integrity follow-up integrated: delivery manifests may declare
  `receiptHash`, `deliverables verify` reports the actual receipt hash, and
  `ledger validate` fails mismatches through `receipt.hash` plus the
  `receipt-hashes` check without adding any repair or write behavior.
